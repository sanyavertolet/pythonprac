Welcome to MUD’s documentation!¶

Условие задачи¶

  • Требуется реализовать простейший multi-user dungeon;

  • Имеется поле 10х10 клеток (по каждой оси нумерация с 0 по 9);

  • В каждой клетке может находиться 0 или более монстров, у каждого монстра есть имя и число очков здоровья (hp - hit points);

  • По полю ходит игрок; попав на клетку с монстром, он может его атаковать, нанося урон (списывая очки здоровья);

  • В начале игры игрок появляется в случайной клетке (0, 0);

  • Клетка с координатами (0, 0) находится в левом верхнем углу поля;

  • Настройка поля и игровой процесс организованы при помощи командной строки (readline/shlex/cmd).

Список команд¶

add monster name <имя монстра> hp <число очков здоровья> coords <X> <Y>¶

Добавить в клетку с заданными координатами монстра с заданным именем и числом очков здоровья. Если в этой клетке уже есть монстр с таким именем, его число очков здоровья меняется на новое.

Примеры:

  • add monster name Gnoll hp 23 coords 5 7

  • add monster name "War Troll" hp 100 coords 4 2

show monsters¶

Вывести про каждого монстра на отдельной строке информацию: его имя, координаты, число очков здоровья.

Примеры:

  • War Troll at (4 2) hp 100

move <направление>¶

Подвинуть игрока на одну клетку в заданном направлении, если это возможно с учетом границ поля.

Возможные направления:

  • up - вверх;

  • down - вниз;

  • left - влево;

  • right - вправо.

Примеры:

  • move up

attack <имя монстра>¶

Атаковать монстра с заданным именем, находящегося в той же клетке, где игрок. Атака отнимает у монстра 10 очков здоровья.

Примеры:

  • attack monstername

Автодополнение¶

Автодополнение работает:

  • для основного имени команды;

  • для направления в команде move;

  • для имени монстра, при использовании команды attack
